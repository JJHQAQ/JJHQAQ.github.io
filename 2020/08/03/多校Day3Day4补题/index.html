HDU 6797 Tokitsukaze and Rescue

题目大意

原题链接：http://acm.hdu.edu.cn/showproblem.php?pid=6797
给你一张图，起点终点，问至多删除k条边，最短路最长是多少。

解决

当时比赛的时候看走眼了，没看到k最大是5，所以直接搜就可以。
每次随便取一条最短路，然后枚举删除其中一条边，搜索回溯，删除k条边后记录答案，取答案最大值。

这样为啥是对的呢？
首先从时间复杂度上考虑，因为是随机的数据，所以最短路不可能特别长，所以搜索算法是可以接受的。

其二从正确性上来考虑，为啥是枚举随便一条最短路，而不是所有最短路呢？
事实证明，搜索所有的最短路会T。至于它为啥是对的，可以这样考虑，对于任意两条最短路径，如果路径上有某一部分是重叠的边，那么枚举哪条最短路都一定会枚举到该边，而对于那些相对独立的边，那么不管枚举那一条路径，总的最短路径长度都是不变的，下一次或者下几次的迭代一定会搜到之前没搜到的最短路径上的边。所以每次只搜索一条最短路径就好了。

顺便，该题用堆优化的dij的话会T,用普通的n^2dij就没事

HDU 6798 Triangle Collision

题目大意

原题链接: http://acm.hdu.edu.cn/showproblem.php?pid=6798
给一个三角形的框，一个点的初始位置和初始速度方向，点在框内，不断运动，碰到框的时候会反弹，满足入射角等于出射角，速度不变，
最后问，第k次碰撞是在什么时间。

解决

其实并不是补题，这个是当时我在考场上做出来的。。。。感觉像是个披着计算几何外套的脑洞题，害。

具体做法就是，将弹射换一个角度想，不是小球弹走，而是整个三角行沿弹射边翻折，所以就转化为，一个点，在一个铺满三角形地砖的平面内沿直线移动，问遇到的第k条边距离起点多长。

然后就相对好做了，总的来说就是，给你一个长度，你总是能O(1)的算出这个长度是经过了几条边，然后二分答案即可。

Code

HDU 6808 Go Runnind

题目大意

原题链接：http://acm.hdu.edu.cn/showproblem.php?pid=6808

解决

题目虽然讲的挺乱的，但是简化出来就比较简单。就等于是给你一堆第一象限的点，你可以用斜率为1或者-1的直线去覆盖他们，问最少需要多少条直线。

然后仔细一看就是个二分图匹配的板子题，就是说。。。将点做边，把边当点，然后跑一遍二分图匹配，最大匹配就等于最小点覆盖，最小点覆盖就显然是我们要求的答案。

这题我抄的HK板子，感觉匈牙利会T，但是隔壁黄大仙他们队的匈牙利就过了，而且跑得比我们还快，惊呆了QAQ。

结果赛后我又把我的改成匈牙利试了试，还真是T的。。。。。直接哭死。
